# AI 시대의 C언어 교육 패러다임 전환: 코드 작성에서 실행 원리 이해 중심으로

## 초록

생성형 인공지능(Generative AI)과 대규모 언어모델(Large Language Models, LLMs)의 발전은 프로그래밍 교육의 패러다임을 빠르게 변화시키고 있다. 특히 LLM은 문법적으로 올바른 C 프로그램을 높은 수준으로 생성할 수 있으며, 이러한 변화는 기존 C언어 교육의 핵심 목표였던 코드 작성 능력(code writing skill)의 교육적 의미를 재검토하도록 요구한다. 그러나 AI가 코드를 생성할 수 있다고 해서 컴퓨터의 동작 원리까지 이해하는 것은 아니다. 본 연구는 AI 활용 교수법을 제안하는 것이 아니라, **AI 시대에 C언어 교육의 목적 자체를 재정의**하는 교육 철학을 제안한다.

본 연구에서는 기존의 **Code-Centric Learning**을 **Execution-Centric Learning**으로 전환할 것을 제안한다. 이를 위해 C언어를 단순한 프로그래밍 언어가 아니라 **Computational Execution을 가장 투명하게 보여주는 창(Window)**으로 정의한다. 학생들은 C언어를 이용하여 메모리 표현, 함수 호출, 스택과 힙, 포인터, 컴파일 및 프로그램 실행 과정을 하나의 통합된 실행 모델(Computational Execution)로 이해해야 한다. 본 연구는 AI 시대에도 C언어가 여전히 핵심 교과목이어야 하는 교육적 근거를 제시하고, 실행 원리 중심의 새로운 교육 목표와 평가 방향을 제안한다.

**주제어:** AI 교육, C언어 교육, 프로그래밍 교육, Computational Execution, Execution-Centric Learning

# 1. 서론

최근 생성형 AI와 LLM의 발전은 프로그래밍 교육 환경을 근본적으로 변화시키고 있다. ChatGPT, Claude, Gemini와 같은 AI 시스템은 자연어 설명만으로도 문법적으로 올바른 프로그램을 생성할 수 있으며, 코드 수정과 디버깅까지 지원하고 있다. 이러한 변화는 "학생들이 여전히 C언어를 배워야 하는가?"라는 질문을 넘어, **"AI 시대에 C언어 교육의 목적은 무엇인가?"**라는 보다 근본적인 질문을 제기한다.

기존의 C언어 교육은 변수, 제어문, 함수, 포인터 등의 문법을 학습하고 이를 이용하여 프로그램을 작성하는 능력을 교육의 핵심 목표로 삼아 왔다. 그러나 AI가 코드 생성 능력을 상당 부분 대체할 수 있는 현재의 환경에서는 코드 작성 능력만을 중심으로 하는 교육은 그 교육적 타당성을 재검토할 필요가 있다.

본 연구는 AI 시대에도 C언어의 중요성이 감소하지 않는다고 주장한다. 다만 **C언어를 가르치는 이유가 달라져야 한다**고 본다. 즉, C언어는 더 이상 프로그램을 작성하기 위한 언어가 아니라, **컴퓨터 내부에서 프로그램이 어떻게 실행되는지를 이해하기 위한 가장 직접적인 교육 도구**로 재정의되어야 한다.

이러한 관점에서 본 연구는 다음과 같은 연구 질문을 제시한다.

> **AI 시대에도 왜 학생들은 C언어를 학습해야 하는가?**

본 연구의 답은 명확하다.

> **AI 시대에 C언어는 코드를 작성하기 위해 배우는 것이 아니라, 컴퓨터의 메모리와 실행 원리를 이해하기 위해 학습한다.**

이를 위해 본 연구에서는 **Computational Execution**을 C언어 교육의 새로운 핵심 학습 목표(Core Learning Objective)로 제안한다. Computational Execution은 프로그램이 메모리에 적재되고, 함수가 호출되며, 데이터가 이동하고, CPU가 명령을 수행하여 결과를 생성하기까지의 전체 실행 과정을 의미한다. C언어는 이러한 실행 과정을 다른 고급 언어보다 가장 투명하게 표현할 수 있기 때문에 교육적으로 독보적인 가치를 가진다.

따라서 본 연구는 AI 활용 교육 사례를 제안하는 것이 아니라, **AI 시대의 C언어 교육 목적을 코드 작성(Code-Centric)에서 실행 원리 이해(Execution-Centric)로 전환하는 새로운 교육 철학**을 제안하는 것을 목표로 한다.

# 2. 관련 연구

프로그래밍 교육에 관한 기존 연구는 크게 세 가지 방향으로 진행되어 왔다.

첫째, **프로그래밍 문법과 문제 해결 능력**을 향상시키기 위한 교육 방법에 관한 연구이다. 이러한 연구들은 변수, 제어문, 함수, 자료구조 등의 문법을 효과적으로 학습시키고, 이를 활용한 알고리즘 구현 능력을 향상시키는 데 초점을 두었다. 특히 C언어 교육에서는 포인터와 동적 메모리 관리와 같은 난이도 높은 개념을 효과적으로 지도하기 위한 다양한 교수법이 제안되었다.

둘째, **컴퓨팅 사고(Computational Thinking)** 중심의 교육 연구이다. Wing 이후 컴퓨팅 사고는 문제를 분석하고 추상화하며 알고리즘으로 해결하는 능력으로 정의되었으며, 프로그래밍은 이러한 사고를 실현하는 수단으로 인식되어 왔다. 그러나 대부분의 연구는 문제 해결 과정에 집중하였으며, 프로그램이 실제 컴퓨터 내부에서 어떻게 실행되는지에 대한 이해는 상대적으로 제한적으로 다루어졌다.

셋째, 최근에는 **생성형 AI를 활용한 프로그래밍 교육**에 관한 연구가 활발히 진행되고 있다. 대부분의 연구는 AI를 활용한 코드 생성, 자동 피드백, 학습 지원, 개인 맞춤형 튜터링 등의 효과를 분석하고 있으며, AI를 새로운 교육 도구로 활용하는 방안을 제안하고 있다.

그러나 기존 연구들은 공통적으로 **AI를 어떻게 교육에 활용할 것인가**에 초점을 두고 있으며, **AI 시대에 C언어를 왜 가르쳐야 하는가**라는 교육 목적 자체를 재정의한 연구는 아직 부족하다.

본 연구는 이러한 기존 연구와 차별화된다. AI를 새로운 교육 도구로 활용하는 방법을 논의하는 것이 아니라, **C언어 교육의 존재 이유를 재정의**한다는 점에서 기존 연구와 구별된다. 특히 C언어를 **"Computational Execution을 가장 투명하게 보여주는 창(C as a Window into Computational Execution)"**으로 정의하고, **Execution-Centric Learning**이라는 새로운 교육 패러다임을 제안한다는 점이 본 연구의 핵심적인 학술적 기여이다.

# 3. AI 시대의 C언어 교육 목적 재정의

생성형 AI의 발전으로 프로그래밍 교육의 환경은 근본적으로 변화하였다. 과거에는 프로그래밍 언어의 문법을 익히고 이를 이용하여 프로그램을 구현하는 능력이 교육의 핵심 목표였다. 그러나 AI는 자연어 설명만으로도 상당 수준의 프로그램을 생성할 수 있으며, 코드의 수정과 최적화까지 지원하고 있다. 이러한 변화는 **"무엇을 가르칠 것인가"보다 "왜 가르칠 것인가"**에 대한 재검토를 요구한다.

본 연구는 AI 시대에도 C언어의 중요성은 유지된다고 본다. 그러나 그 이유는 더 이상 프로그램을 작성하기 위해서가 아니다. C언어의 가장 큰 교육적 가치는 **프로그램이 컴퓨터 내부에서 어떻게 실행되는지를 가장 직접적으로 설명할 수 있다는 점**에 있다.

예를 들어 다음과 같은 C 문장을 생각해 보자.

```c
a = b + c;
```

기존 교육에서는 이 문장을 변수와 연산자의 사용 예제로 설명한다. 그러나 실행 관점에서는 전혀 다른 의미를 가진다.

* 메모리에서 `b`와 `c`의 값을 읽는다.
* CPU가 두 값을 연산한다.
* 결과를 `a`가 위치한 메모리에 저장한다.

즉, 하나의 C 문장은 **메모리의 상태 변화와 계산의 실행 과정**을 표현한 것이다. 이러한 관점에서 C언어는 문법을 배우는 대상이 아니라 **컴퓨터의 실행 원리를 이해하는 도구**가 된다.

따라서 AI 시대의 C언어 교육 목표는 다음과 같이 재정의될 수 있다.

> **학생은 코드를 작성하는 사람이 아니라, 프로그램이 컴퓨터 내부에서 어떻게 실행되는지를 설명할 수 있는 사람이 되어야 한다.**

이러한 교육 목표의 변화는 Code-Centric Learning에서 Execution-Centric Learning으로의 패러다임 전환을 의미한다.

# 4. C as a Window into Computational Execution

본 연구는 **Computational Execution**을 AI 시대 C언어 교육의 핵심 학습 목표(Core Learning Objective)로 제안한다.

Computational Execution은 프로그램이 컴퓨터에서 실행되는 전 과정을 의미한다. 여기에는 프로그램의 컴파일, 메모리 적재, 함수 호출, 데이터 이동, 연산 수행, 메모리 변경, 프로그램 종료까지의 모든 과정이 포함된다.

기존의 C언어 교육에서는 이러한 개념을 각각 독립적으로 학습하였다.

* 변수
* 함수
* 포인터
* 배열
* 스택
* 힙
* 컴파일

그러나 실제 컴퓨터에서는 이러한 요소들이 분리되어 동작하지 않는다. 하나의 프로그램 실행 과정 안에서 상호 연관되어 동작한다.

예를 들어 함수 호출은 단순히 함수를 실행하는 것이 아니라,

* 호출 스택(Stack)에 새로운 프레임을 생성하고,
* 매개변수를 전달하며,
* 지역변수를 메모리에 할당하고,
* 반환 주소를 저장한 후,
* 함수 종료와 함께 메모리를 회수하는 일련의 실행 과정이다.

마찬가지로 포인터 역시 문법적 요소가 아니라 **메모리 주소를 이용하여 데이터가 어떻게 참조되고 이동하는가를 설명하는 실행 모델의 일부**이다.

따라서 학생들은 포인터 자체를 암기하는 것이 아니라,

> **"프로그램이 실행되는 동안 메모리가 어떻게 변화하는가?"**

를 설명할 수 있어야 한다.

이러한 관점에서 본 연구는 C언어를 다음과 같이 정의한다.

> **C is a Window into Computational Execution.**

즉, C언어는 Computational Execution 자체가 아니라, 이를 가장 투명하게 관찰하고 이해할 수 있도록 하는 교육적 창(Window)이다.

# 5. Execution-Centric Learning Framework

본 연구에서는 AI 시대의 C언어 교육을 위한 새로운 교육 프레임워크를 제안한다.

기존 교육은 프로그램을 작성하는 과정에 초점을 맞추었다.

**입력 → 문법 학습 → 코드 작성 → 실행 결과 확인**

반면 Execution-Centric Learning은 실행 과정을 이해하는 것을 중심으로 구성된다.

**문제 이해 → 실행 과정 예측 → 메모리 변화 분석 → 실행 결과 설명 → 코드 개선**

이 과정에서 AI는 코드 생성 도구로 활용될 수 있다. 학생은 AI가 생성한 코드를 그대로 사용하는 것이 아니라,

* 메모리가 어떻게 구성되는가?
* 함수 호출은 어떻게 이루어지는가?
* 포인터는 무엇을 가리키는가?
* 실행 과정에서 데이터는 어떻게 이동하는가?

를 분석하고 설명해야 한다.

이러한 교육 방식에서는 코드 작성 자체보다 실행 원리를 설명하는 능력이 더욱 중요하다.

### 표 1. 기존 C언어 교육과 제안하는 교육 프레임워크의 비교

| 구분     | 기존 교육      | 제안하는 교육                 |
| ------ | ---------- | ----------------------- |
| 교육 목적  | 코드 작성 능력   | 실행 원리 이해                |
| 중심 개념  | 문법(Syntax) | Computational Execution |
| 학습 방식  | 구현 중심      | 실행 과정 중심                |
| AI의 역할 | 코드 생성      | 실행 분석 대상                |
| 학생의 역할 | 프로그래머      | 실행 원리 설명자               |
| 평가     | 프로그램 구현    | 실행 과정 설명 및 분석           |

본 연구는 다음과 같은 교육 철학을 제안한다.

> **AI 시대에 C언어는 코드를 작성하기 위해 배우는 것이 아니라, 컴퓨터의 메모리와 실행 원리를 이해하기 위해 학습한다.**

이 철학은 단순히 C언어 교육의 변화를 의미하지 않는다. AI 시대의 컴퓨터공학교육은 **구현 능력(Implementation)** 중심에서 **실행 원리 이해(Understanding Computational Execution)** 중심으로 전환되어야 함을 의미한다.

# 6. AI 시대의 평가 방법

교육 목표가 변화하면 평가 방법 역시 변화해야 한다. 기존의 C언어 평가는 주로 문법 활용 능력과 프로그램 구현 능력을 중심으로 이루어졌다. 대표적인 평가 문항은 "주어진 문제를 해결하는 프로그램을 작성하시오."와 같은 형태이다.

그러나 생성형 AI는 이러한 구현 과제를 높은 수준으로 수행할 수 있다. 따라서 AI 시대에는 코드 작성 능력만으로 학생의 학습 성과를 평가하는 것은 적절하지 않다.

본 연구에서는 **Computational Execution**에 대한 이해를 평가의 중심으로 제안한다. 학생은 프로그램을 작성하는 능력뿐 아니라, 프로그램이 실행되는 과정에서 컴퓨터 내부에서 발생하는 변화를 설명할 수 있어야 한다.

예를 들어 다음과 같은 평가가 가능하다.

* 주어진 코드가 실행될 때 메모리의 변화를 설명하시오.
* 함수 호출 과정에서 스택 프레임이 어떻게 생성되고 제거되는지를 설명하시오.
* 포인터가 가리키는 메모리의 상태를 실행 순서에 따라 설명하시오.
* AI가 생성한 코드의 실행 과정을 분석하고 오류 가능성을 설명하시오.

이러한 평가는 단순한 코드 생성 능력이 아니라 실행 원리에 대한 이해를 측정할 수 있다.

또한 AI는 평가의 대상이 아니라 **학습 도구**로 활용될 수 있다. 학생은 AI가 생성한 코드를 분석하고, 실행 과정의 타당성을 검증하는 역할을 수행한다. 이는 AI와 협업하는 미래의 소프트웨어 개발 환경에도 부합하는 교육 방식이다.

향후에는 실행 과정에 대한 이해 수준을 객관적으로 측정할 수 있는 평가 루브릭(rubric)의 개발과 교육 효과 검증이 필요하다.

# 7. 결론

생성형 AI의 발전은 프로그래밍 교육의 환경을 변화시키고 있으며, C언어 교육 역시 새로운 관점에서 재해석될 필요가 있다. 본 연구는 AI 활용 방법을 제안하는 것이 아니라, **AI 시대에 C언어 교육의 목적을 재정의하는 교육 철학**을 제안하였다.

기존의 C언어 교육은 코드 작성 능력을 중심으로 구성되어 왔다. 그러나 AI가 프로그램 생성 능력을 지원하는 현재의 환경에서는 학생이 반드시 갖추어야 할 핵심 역량이 달라지고 있다. 이제 학생은 프로그램을 작성하는 사람을 넘어, **프로그램이 컴퓨터 내부에서 어떻게 실행되는지를 설명할 수 있는 사람**이 되어야 한다.

이를 위해 본 연구는 **Computational Execution**을 AI 시대 C언어 교육의 새로운 핵심 학습 목표로 제안하였다. 또한 Code-Centric Learning에서 Execution-Centric Learning으로의 교육 패러다임 전환을 제시하였다.

본 연구에서 제안한 교육 철학은 C언어 교육에 국한되지 않는다. 프로그램의 실행 원리를 이해하는 능력은 운영체제, 컴퓨터구조, 컴파일러, 임베디드 시스템, 인공지능 시스템 등 컴퓨터공학 전반을 연결하는 핵심 역량이다. 이러한 의미에서 C언어는 단순한 프로그래밍 언어가 아니라, **Computational Execution을 가장 투명하게 보여주는 교육 도구**로 이해될 수 있다.

본 연구의 핵심 메시지는 다음과 같다.

> **AI 시대에 C언어는 코드를 작성하기 위해 배우는 것이 아니라, 컴퓨터의 메모리와 실행 원리를 이해하기 위해 학습한다.**

즉,

> **C is a Window into Computational Execution.**

향후 연구에서는 실제 수업에 본 교육 프레임워크를 적용하여 학습 효과를 분석하고, Computational Execution에 대한 이해 수준을 객관적으로 측정할 수 있는 평가 모델을 개발할 필요가 있다.
