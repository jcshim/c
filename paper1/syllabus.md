# 논문 적용 전후 강의계획서 차이 요약

| 주차 | 적용 전 (Code-Centric) | 적용 후 (Execution-Centric) |
|---|---|---|
| 1 | 설치 + 문법 복습 | + 변수의 메모리 저장 방식 소개 |
| 2 | 반복문 문법 | + 반복 실행 중 메모리 값 변화 추적 |
| 3 | 배열/함수/시뮬레이션 구현 | + 함수 호출 시 스택 프레임 생성·소멸 설명 |
| 4 | qsort/이진탐색 구현 | + 정렬 중 메모리 스왑, 함수포인터 실행 흐름 설명 |
| 5 | 라이브러리 생성법 | + 컴파일-링크 시 메모리 적재 원리 설명 |
| 6 | gcc 컴파일/실행법 | + 컴파일 단계별(전처리~링크) 산출물 분석 |
| 7 | Makefile 작성법 | + 빌드~실행파일 생성 원리 설명 |
| 8 | 구현 중심 시험 | + 실행 과정/메모리 설명 문항 추가 |
| 9-10 | SQLite CRUD 구현 | + 포인터로 DB 핸들 참조하는 구조 설명 |
| 11-12 | AI로 게임 코딩, 결과 확인 | AI 코드의 구조체/포인터/메모리 변화를 **분석 대상**으로 전환 |
| 13-14 | 프로젝트 구현 | + 설계 단계 메모리 구조 계획, AI 코드 검증 절차 추가 |
| 15 | 구현 결과 평가 | 구현 + **실행 과정 설명 능력** 평가로 확장 |

**한 줄 요약**
- **적용 전**: "코드를 작성할 수 있는가"를 중심으로 구성
- **적용 후**: "코드가 실행될 때 컴퓨터 내부에서 무슨 일이 일어나는지 설명할 수 있는가"가 매 주차에 추가됨
- **가장 큰 변화**: AI 활용 주차(9-12주)에서 AI가 "코드를 대신 짜주는 도구"에서 "학생이 분석·검증해야 할 대상"으로 역할이 바뀜
