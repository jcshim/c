### 1) 단일 연결리스트 삽입의 시간 복잡도

단일 연결리스트에서 **머리(front/head) 삽입** 연산의 시간 복잡도는?
A. (O(n))
B. (O(\log n))
C. (O(1))
D. (O(n \log n))
**정답:** C
**해설:** 새 노드를 만들고 `new->next = head; head = new;`만 수행하므로 상수 시간.

### 2) 원형 큐의 공백/포화 조건

배열 길이가 `N`인 원형 큐에서 **한 칸 비우는 방식**을 사용한다. `front`는 **첫 원소의 위치**, `rear`는 **마지막 원소의 다음 위치**를 가리킨다. 올바른 조건은?
A. 공백: `front == rear`, 포화: `front == rear + 1`
B. 공백: `front == rear`, 포화: `(rear + 1) % N == front`
C. 공백: `(rear + 1) % N == front`, 포화: `front == rear`
D. 공백: `front == rear == -1`, 포화: `rear == N-1`
**정답:** B
**해설:** 한 칸 비우는 관례에서 공백은 동일, 포화는 다음 위치가 `front`와 같을 때.

### 3) 스택을 이용한 괄호 검사

아래 문자열에 대해 스택으로 괄호 짝을 검사한다. **올바른 판정**은?
입력: `"{[()()]()}"`
A. 불일치
B. 중복 닫힘
C. 누락된 닫힘
D. 일치
**정답:** D
**해설:** 여는 괄호는 push, 닫는 괄호는 pop 비교. 끝나고 스택이 비면 일치.

### 4) 이진 트리 순회 결과

다음 이진 트리에 대해 **중위(Inorder) 순회** 결과는?

```
        F
       / \
      B   G
     / \   \
    A   D   I
       / \  /
      C  E H
```

A. F B A D C E G I H
B. A B C D E F G H I
C. A B C D E F G H I (레벨순)
D. A B D C E F G H I
**정답:** B
**해설:** 중위 = 좌·루트·우. 각 서브트리에 적용하면 정렬된 순서가 된다.

### 5) 해시 테이블(선형 조사) 탐사 순서

크기 7의 해시 테이블, 해시 함수 `h(k) = k % 7`, **선형 조사(linear probing)** 사용. 빈 테이블에 순서대로 키 `14, 21, 28, 5`를 삽입할 때 `5`의 최종 위치는?
A. 인덱스 5
B. 인덱스 0
C. 인덱스 1
D. 인덱스 2
**정답:** C
**해설:** `14→0`, `21→0 충돌→1`, `28→0 충돌→2`, `5→5`는 비었으므로 바로 **5** 같지만… 잠깐! 앞 삽입들을 다시 보면 0,1,2가 찼고 5는 비어 있습니다. 따라서 정답은 **A(5)**여야 해? 아닙니다—문제 의도 점검: 앞 세 키가 0,1,2를 차지하고 있어도 5는 충돌 없이 **바로 5**에 들어갑니다.
→ **정답 수정: A(인덱스 5)**
(검산: `14%7=0`, `21%7=0→1`, `28%7=0→2`, `5%7=5`)
