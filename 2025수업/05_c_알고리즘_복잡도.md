다음은 **시간 복잡도 (O(n))**(선형 시간)의 핵심만 쏙 정리한 설명입니다.

* **뜻**: 입력 크기 (n)이 커질수록 실행 시간이 **(n)에 비례해서** 늘어난다는 의미입니다.
  예) 길이 (n) 배열을 한 번 쭉 훑는 알고리즘.

* **왜 선형?** 한 원소마다 상수 작업을 1번씩 하니, 총 작업 수가 대략 (c \times n) → 빅오 표기에서는 상수 (c)를 무시해 (O(n)).

* **전형적인 예**

  * 배열 최대값 찾기, 합계 구하기, 특정 값 포함 여부(정렬 X, 한 바퀴 탐색), 연결리스트 전체 순회.
  * 문자열을 처음부터 끝까지 보며 조건 검사.

* **간단 C 예시 (양수 개수 세기)**

  ```c
  int count_positive(int *a, int n) {
      int cnt = 0;
      for (int i = 0; i < n; i++) { // n번 반복 → O(n)
          if (a[i] > 0) cnt++;
      }
      return cnt;
  }
  ```

* **헷갈리기 쉬운 점**

  * 루프가 **하나**라도, 내부에서 비싼 연산이 없다면 보통 (O(n)).
  * 두 번 훑어도 (O(n) + O(n) = O(n)) (상수 배수는 무시).
  * 중첩 이중 루프로 모든 쌍을 보면 (O(n^2))이 되어 **선형이 아님**.

* **공간 복잡도는 별개**

  * 위 예시는 추가 배열을 만들지 않으므로 공간 (O(1)).
  * 입력 크기만큼 별도 버퍼를 쓰면 공간 (O(n)).

요약: **입력을 한 번(또는 몇 번)만 “쭉” 훑는** 유형이면 시간 복잡도는 보통 **(O(n))** 입니다.
