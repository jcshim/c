# ✅ **C 언어 실력 향상을 위한 5단계 로드맵**

## **1단계. 기초 문법을 ‘정확하게’ 다지는 단계**

> “대충 아는 상태”는 절대 실력이 안 늘어요.
> **정확히 아는지 확인하는 문제**를 반드시 풀어야 합니다.

학생이 다음 개념을 *실수 없이* 쓸 수 있어야 합니다.

* 변수 / 배열 / 포인터 / 함수
* 구조체(struct)
* 동적 메모리(malloc/free)
* 파일 입출력
* 문자열(char[], strcpy)

👉 **추천 실습**

* 문자열 뒤집기
* 구조체 배열 정렬
* 파일에서 숫자 읽어 평균 계산
* malloc으로 동적 배열 만들기

이 정도를 *스스로 코딩*할 수 있으면 1단계 통과입니다.

---

## **2단계. “오류 해결” 경험을 쌓기**

C는 포인터, 메모리 오류가 많아서
**디버깅 연습 없이는 절대 성장하지 않습니다.**

학생에게 이런 문제를 일부러 주면 실력이 빠르게 늡니다.

* 잘못된 포인터 접근 → segmentation fault
* 메모리 누수 → free 빠뜨림
* 배열 범위 초과 → 쓰레기 값 발생
* 문자열 끝 ‘\0’ 누락 → 이상한 출력

👉 **추천 툴**

* Visual Studio Debugger
* gdb
* Valgrind (메모리 오류 잡는 도구)

> **버그를 100개 잡으면 C 언어 실력이 폭발적으로 오른다.**

---

## **3단계. “작은 프로젝트”를 반드시 해보기**

문법 공부만으로는 절대 실력이 안 늡니다.
반드시 “작품”을 만들어야 합니다.

### ✔ 추천 프로젝트 (난이도 낮지만 교육효과 최고)

* **미니 계산기 프로그램** (동적메모리 + 함수)
* **학생 성적 관리(구조체 + 파일)**
* **텍스트 기반 게임 만들기 (예: 숫자 맞추기)**
* **주소록 관리 프로그램**
* **스택/큐 직접 구현**

학생이 **직접 고민해서 완성하는 경험**이 가장 중요합니다.

---

## **4단계. 고급 주제로 자연스럽게 확장**

기본기가 잡힌 후에는
아래 주제를 하나씩 맛보면 실력이 확실히 올라갑니다.

* 포인터 더 깊게 (이중 포인터, 함수 포인터)
* 동적 자료구조 직접 구현

  * 연결리스트
  * 스택 / 큐 / 트리
* 메모리 모델

  * 스택 / 힙 / 전역영역
* 시간 복잡도(알고리즘)
* 간단한 네트워크 프로그래밍(TCP/UDP 기초)

학생이 “이게 가능하네?”라고 느끼면 성장 속도가 빨라집니다.

---

## **5단계. 결국 ‘실전’에 가까운 경험 만들기**

**C 언어를 잘한다 = 메모리·포인터 기반 사고가 탄탄하다**는 뜻입니다.
이를 활용하려면 다음 분야로 확장하는 것이 가장 좋습니다.

### ✔ 추천 확장 분야

* **임베디드(마이크로컨트롤러, 아두이노, STM32)**
* **운영체제 실습(리눅스 시스템 호출)**
* **알고리즘 문제풀이(BOJ 100~200문제)**
* **간단한 드라이버 실습**

특히 컴공 1학년 학생이라면
**아두이노로 센서 + 모터 제어**를 해보면
C 언어에 대한 이해가 급격히 깊어집니다.
