# 🎉 **1. 도입: ‘함수는 마법 주문이다!’ 시연(10분)**

학생들이 어려워하는 이론을 말로 설명하지 말고, **보이는 시연**으로 시작합니다.

* add()를 호출하면 화면에 숫자가 “뚝!” 나오는 모습
* 지역 변수는 show() 안에서만 살아있다가 사라지는 애니메이션
* factorial(5) 호출하면 스택이 5→4→3→2→1 순서로 쌓였다가 돌아오는 그림

👉 “오늘은 이걸 **외우는 시간**이 아니라, **함수 축제**를 해보는 시간입니다”라고 선언

# 🎯 **2. 함수 빙고 게임(20분)**

이론을 외우게 하지 말고 **빙고판** 형태로 주면 흥미도 상승!

빙고칸 예시:

* 함수 선언 vs 정의
* 매개변수/인자
* Call by Value
* 지역/전역 변수
* 반환형
* 재귀의 ‘종료 조건’
* 스택 프레임
* 헤더파일 역할

교수님이 예시 코드를 하나씩 보여주며 “이 코드에서 해당 개념이 어디 있나?” 찾기 게임.

👉 이론이 아니라 **퍼즐**이 됨 → 저절로 이해됨.

# ⚡ **3. 실험 미션: “예제 3개를 직접 변형하기”(30분)**

과제가 아니라, **예제를 ‘깨부수고 다시 만드는 경험’**을 준다.

## ✔ 실험 1: add() 변형

* 함수 이름 바꾸기
* 매개변수 개수 늘리기
* 반환형 변경(int → double)
* printf를 함수 안/밖에 넣어보기

👉 선언·정의·반환형·매개변수 전체 복습

## ✔ 실험 2: 지역/전역 변수 혼동 실험

* 변수 이름 고의로 중복
* 함수 안에서 값 바꾼 뒤 main()에 영향이 있는지 확인
* g 값을 printf 여러 위치에 넣고 흐름 확인

👉 scope·lifetime·stack 자연스럽게 이해

## ✔ 실험 3: 재귀를 반복문으로 바꾸는 놀이

* factorial 재귀 → for 반복문 버전 직접 만들어보기
* n이 큰 경우 스택이 어떻게 되는지 그림으로 표시
* base case 없애보면 무슨 일이 일어나는지 직접 보기(무한 재귀)

👉 재귀를 어려워하는 학생이 **직접 비교**하며 이해

# 🧠 **4. “함수 쇼케이스” – 나만의 함수 하나 만들어 발표하기 (25분)**

모든 학생이 단 하나의 함수만 만들면 됨.
주제 자유:

* 나만의 별찍기 함수
* 영어 단어 출력 함수
* 랜덤 숫자 생성 함수
* 점수 평균 함수
* 미니 콘텐츠 출력 함수 등

조건은 단 하나:

> ⭐ “함수의 이름, 매개변수, 반환형을 **왜** 이렇게 정했는지 설명하기.”

이 단계에서 함수의 **헤더 구조**, **반환형**, **인자**가 자연스럽게 복습됨.

# 🎁 **5. 마지막 15분: ‘함수의 비밀 5개’ 퀴즈쇼**

학생들이 가장 헷갈려 하는 개념만 골라 **퀴즈 + 체험**으로 끝맺음.

내용은 아래 5개만:

1. 함수 선언과 정의 차이를 말해보세요
2. Call by Value란?
3. 지역 변수는 언제 사라지나요?
4. 재귀에서 꼭 필요한 한 줄은? (정답: base case)
5. 스택은 어떤 순서로 쌓이고 사라지나요?

정답 맞춘 학생에게 작은 상품(사탕/스티커/초콜릿 등)
→ **수업 분위기가 축제처럼 됨**
